SuccessResponse

class SuccessResponse extends Response

When a rpc call is made, the Server MUST reply with a Response, except for in the case of Notifications. The Response is expressed as a single JSON Object, with the following members:

Properties

string $jsonrpc A String specifying the version of the JSON-RPC protocol. MUST be exactly "2.0". from  Message
int|string $id This member is REQUIRED. It MUST be the same as the value of the id member in the Request Object. If there was an error in detecting the id in the Request object (e.g. Parse error/Invalid Request), it MUST be Null. from  Response
mixed $result This member is REQUIRED on success. This member MUST NOT exist if there was an error invoking the method. The value of this member is determined by the method invoked on the Server.
